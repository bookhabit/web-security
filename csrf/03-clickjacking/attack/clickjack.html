🔍 클릭재킹 공격 원리

1
공격자가 타겟 사이트(기부 페이지)를 투명한 iframe으로 불러옵니다.
2
iframe을 가짜 버튼 위에 정확히 겹쳐 배치합니다.
/* ── 핵심 CSS ── */ .target-iframe { position: absolute; /* 가짜 버튼 위에 절대 위치 */ top: 0; left: 0; width: 100%; height: 100%; opacity: 0; /* ← 완전 투명! 피해자에게 보이지 않음 */ z-index: 10; /* ← 가짜 버튼보다 위에 위치 */ pointer-events: all; /* ← 클릭 이벤트 수신 */ }
3
피해자가 "아이템 받기"를 클릭 → 실제로는 투명 iframe의 "기부하기" 버튼이 클릭됨
4
피해자의 세션 쿠키로 10만원 기부가 완료됩니다.
⚡ 위 "공격 iframe 보기" 버튼을 클릭하면 투명 레이어가 어떻게 겹쳐있는지 확인할 수 있습니다.
빨간 테두리 = 실제 타겟 페이지 위치

🛡️ 방어법: X-Frame-Options & CSP

// PHP에서 헤더 추가 header('X-Frame-Options: DENY'); // 또는 CSP (더 현대적) header('Content-Security-Policy: frame-ancestors \'none\'');

방어 버전: http://localhost:8095/donate-secure.php

방어 버전을 iframe에 넣으려 하면 브라우저가 "이 페이지를 표시할 수 없음"으로 차단합니다.
